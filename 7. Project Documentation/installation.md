# Installation Guide

This guide walks you through getting OptiCrop running locally from scratch, then covers the Docker path.

---

## Prerequisites

- Python 3.12 or later
- pip
- Git
- Docker and Docker Compose (optional)

---

## Local Installation

### 1. Clone the repository

```bash
git clone <repository-url>
cd opticrop
```

### 2. Create a virtual environment

```bash
python -m venv .venv
```

### 3. Activate the virtual environment

**Windows:**
```bash
.venv\Scripts\activate
```

**macOS / Linux:**
```bash
source .venv/bin/activate
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. Configure environment variables

Copy the example file:
```bash
cp .env.example .env
```

Open `.env` and fill in every required value:

| Variable | Description |
|---|---|
| `SECRET_KEY` | Flask session secret — use a long random string in production |
| `CSRF_SECRET_KEY` | CSRF protection secret |
| `ADMIN_USERNAME` | Admin dashboard username |
| `ADMIN_PASSWORD_HASH` | bcrypt hash of the admin password |
| `SUITABILITY_THRESHOLDS` | JSON string defining per-crop agronomic thresholds |
| `ACTIVE_MODEL_PATH` | Path to the serialized model `.pkl` (set after training) |
| `ACTIVE_SCALER_PATH` | Path to the serialized scaler `.pkl` (set after training) |
| `DATASET_PATH` | Path to the training CSV (default: `datasets/crop_recommendation.csv`) |

Optional variables with defaults are listed in `.env.example`.

To generate a bcrypt hash for the admin password:
```python
import bcrypt
print(bcrypt.hashpw(b"your_password", bcrypt.gensalt()).decode())
```

### 6. Train the model

```bash
python scripts/train.py
```

The pipeline ingests the dataset, preprocesses the data, trains nine algorithms, evaluates them, selects the best by F1-weighted score, and writes artifacts to `saved_models/`:

```
saved_models/
├── ExtraTrees_2026-07-01T122126Z.pkl
├── scaler_2026-07-01T122126Z.pkl
└── metadata.json
```

It also writes EDA plots to `app/static/eda/`.

### 7. Update model paths in `.env`

Set `ACTIVE_MODEL_PATH` and `ACTIVE_SCALER_PATH` to the exact filenames generated by the training script:

```ini
ACTIVE_MODEL_PATH=saved_models/ExtraTrees_2026-07-01T122126Z.pkl
ACTIVE_SCALER_PATH=saved_models/scaler_2026-07-01T122126Z.pkl
```

### 8. Start the application

```bash
python wsgi.py
```

Or using the Flask development server:
```bash
python -m flask run
```

The app is available at `http://localhost:5000`.

---

## Docker Installation

If you prefer Docker, skip steps 2–8 above and use Docker Compose instead.

### 9. Docker Compose

```bash
docker-compose up --build
```

Docker Compose builds the image, creates named volumes for persistent data, and starts the container on port 5000. See [docker.md](docker.md) for full details including how to run training inside the container.

---

## Verifying the Installation

Once the app is running, confirm it is healthy:

```bash
curl http://localhost:5000/api/health
# {"status": "ok", "uptime_seconds": 12}
```

Then open `http://localhost:5000` in a browser to see the home page.

---

## Common Issues

**`ACTIVE_MODEL_PATH` not found at startup** — The app starts but logs a warning. Run `python scripts/train.py` and update `.env` with the generated paths.

**`ModuleNotFoundError`** — Make sure the virtual environment is activated and `pip install -r requirements.txt` completed without errors.

**`CSRF_SECRET_KEY` not set** — The app halts on startup with an error identifying the missing variable. Add it to `.env`.
